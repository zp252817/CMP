DVSync 接入方案文档

1. 方案概述

1.1 背景

在滚动场景中，为了优化性能和功耗，需要在抛滑（fling）动画期间动态开启/关闭 VSYNC 的动态刷新率功能。HarmonyOS 提供了 OH_NativeVSync_DVSyncSwitch API 来实现这一功能。

1.2 目标

  • 在用户开始抛滑时启用DVSYNC
  • 在抛滑动画结束时禁用DVSYNC
  • 通过完整的调用链路从 Compose 层到 Native 层实现功能

1.3 参考文档

  • HarmonyOS Native VSync API 文档

3.5 Native C++ 层实现

3.5.1 RenderNodeImplC.cpp

位置: OHRender/OHRender/src/oh/RenderNodeImplC.cpp

内部实现:

namespace {
static OH_NativeVSync* g_vsync_handle = nullptr;
static bool enableDvsync = false;
// 启用/禁用动态 VSYNC
static void EnableDvsyncImpl(bool enable) {
enableDvsync = enable;
EnsureVsyncHandle();
if (g_vsync_handle == nullptr) {
OH_LOG_Print(LOG_APP, LOG_WARN, LOG_DOMAIN, "cmp",
"EnableDvsync: vsync handle is null");
return;
}
// 调用 HarmonyOS Native VSync API 开启/关闭动态 VSYNC
int32_t result = OH_NativeVSync_DVSyncSwitch(g_vsync_handle, enable ? 1 : 0);
if (result != 0) {
OH_LOG_Print(LOG_APP, LOG_WARN, LOG_DOMAIN, "cmp",
"EnableDvsync: DVSyncSwitch failed, result=%{public}d", result);
} else {
OH_LOG_Print(LOG_APP, LOG_DEBUG, LOG_DOMAIN, "cmp",
"EnableDvsync: %{public}s", enable ? "enabled" : "disabled");
}
}
} // namespace
// 导出给其他编译单元使用的接口：启用/禁用动态 VSYNC
void EnableDvsync(bool enable) {
EnableDvsyncImpl(enable);
}

4. 关键 API 说明

4.1 HarmonyOS Native VSync API

函数签名:

int32_t OH_NativeVSync_DVSyncSwitch(OH_NativeVSync* vsync, int32_t enable);

参数说明:- vsync: VSync 句柄，通过 OH_NativeVSync_Create() 创建- enable: 1 表示启用动态 VSYNC，0 表示禁用

返回值:- 0: 成功- 非 0: 失败

使用时机:- 在抛滑动画开始时调用 enable = 1- 在抛滑动画结束时调用 enable = 0

8. 注意事项

8.1 VSync 句柄管理

  • VSync 句柄通过 EnsureVsyncHandle() 确保已创建
  • 如果句柄为 null，函数会安全返回，不会崩溃

8.2 线程安全

  • EnableDvsync() 函数在 Compose 协程中调用
  • VSync 句柄的创建和管理是线程安全的

8.3 平台特定性

  • 此功能仅在 OHOS 平台实现
  • 其他平台（Android、iOS 等）的 enableDvsyncForLTPO() 需要提供空实现或平台特定实现

8.4 性能考虑

  • 动态 VSYNC 的开启/关闭是轻量级操作
  • 在抛滑开始时启用，结束时禁用，避免长期占用资源
